Tony Stark Mode

Vision Lab

맥북프로 16인치에서 가능한 비전 제어 실험을 현실 기준으로 정리한 패널입니다. 일반 웹캠 기반 Head Pointer는 가능하고, Face ID용 적외선/TrueDepth 카메라 활용이나 독립 이중 포인터 분리는 공개 macOS 기준으로 어렵습니다.

가능

웹캠 기반 얼굴 / 머리 추적

macOS 접근성 Head Pointer와 얼굴 표정 기반 Alternate pointer actions 조합은 현실적으로 쓸 수 있습니다.

  • 단일 시스템 포인터를 얼굴 / 머리로 제어
  • 표정으로 클릭, 더블클릭, 드래그 트리거
  • 카메라 선택 가능
어려움

Face ID / IR / TrueDepth 활용

맥북프로 16인치 공개 사양 기준으로 Face ID, TrueDepth, 적외선 카메라 하드웨어가 노출되지 않습니다.

  • 생체 인증은 Touch ID 중심
  • 공개 하드웨어 문서에 Face ID 없음
  • IR 기반 독자 제어 시스템 전제는 부적합
제한

독립 이중 포인터

외장 모니터와 맥북 내장 화면을 서로 다른 입력 시스템으로 완전히 분리하는 건 공개 macOS 기능만으로는 어렵습니다.

  • Head Pointer는 시스템 포인터 하나를 움직임
  • 마우스/트랙패드 입력 시 Head Pointer는 일시정지 후 재개
  • 결론적으로 “번갈아 제어”는 가능하지만 “완전 분리”는 어려움
현실적인 아키텍처

3가지 운영안

1안. 순정 macOS 접근성만 사용 외장 모니터 포함 단일 포인터를 Head Pointer로 움직이고, 표정 클릭을 붙입니다. 구현 난도가 가장 낮습니다.
2안. 외장 모니터 위 웹캠 추가 외장 모니터를 볼 때도 시선축이 맞도록 외장 카메라를 Head Pointer용으로 지정합니다. 실제 사용성은 이 구성이 가장 낫습니다.
3안. Lid Angle Sensor 실험 포함 맥북 화면 각도에 따라 감도 전환, 모드 전환, 앱 실행을 붙입니다. 다만 비공식 센서 접근이라 업데이트 리스크가 있습니다.
비공식 센서

Lid Angle Sensor

화면 기울기 기반 제어는 가능성이 있지만 공식 API가 아니라 실험 레벨로 다뤄야 합니다.

  • IOHIDManager 기반 숨겨진 lid angle 센서 접근
  • macOS 업데이트 / 모델별 차이로 깨질 수 있음
  • 프로토타입과 개인 프로젝트엔 적합
제품 통합 방향

Office Agent와 합치는 방식

  • 메뉴바 펫 모니터는 시스템 상태 표시 담당
  • Vision Lab은 가능한 제어 방식과 가이드를 정리하는 실험실 역할
  • 실제 포인터 제어는 macOS 접근성으로 넘기고 앱은 상태/설정 허브 역할 수행
  • 장기적으로는 lid angle, 외장 카메라, 앱 자동 전환 시나리오를 이 창에서 실험
